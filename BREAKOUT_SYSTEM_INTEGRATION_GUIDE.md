# 🚀 完整突破交易系统 - 部署指南

## 系统架构

```
【发现阶段】
   breakout_scanner.py
   └─ 扫描全市场 5244 只股票
   └─ 发现当天放量突破信号
   └─ 推送到飞书

         ↓

【追踪阶段】
   breakout_decision_tree.py
   └─ 追踪昨天的突破信号
   └─ 5日内判断买入时机
   └─ 路径A：放量涨 → ✅ 买入信号
   └─ 路径B：缩量跌 → ⏳ 等待回踩
   └─ 路径C：放量跌 → ❌ 放弃
   └─ 推送到飞书

         ↓

【整合系统】
   integrated_breakout_system.py
   └─ 自动连接扫描→追踪
   └─ 一键运行完整流程
   └─ 生成整合报告
```

## 📦 部署内容

### 核心脚本（3个）

| 脚本 | 功能 | 运行时间 | 输出 |
|------|------|--------|------|
| **breakout_scanner.py** | 全市场扫描 MA20/60/120 突破 | 收盘后 17:30-40 | 新发现的突破信号 |

## 🎯 买入信号判断标准

### 绿灯信号（🟢 BUY）

✅ **路径A：放量涨** — 突破确认，可介入
- 放量突破后，次日或多日内放量上涨
- 质量：**标准**

✅ **路径A：缩量涨** — 惜售信号，优质买点
- 放量突破后，缩量上涨（说明筹码集中）
- 质量：**优质**

✅ **路径B：回踩企稳** — 最优买点
- 放量突破后回踩
- 出现缩量十字星/小阳线
- 收盘站上MA20
- 质量：**回踩确认（最优）**

### 黄灯信号（🟡 PULLBACK_WAIT）

⏳ **路径B：缩量回踩中**
- 放量突破后缩量下跌
- 但未破MA20
- 继续等待企稳信号

### 红灯信号（🔴 ABANDON）

❌ **路径C：放量跌** — 假突破，放弃
- 放量突破后放量下跌
- 说明之前是虚假突破

❌ **连续跌破MA20** — 止损
- 突破后再次跌破MA20
- 说明支撑无效

## 💡 操作纪律

```
【操作流程】

1️⃣ 发现 (扫描日)
   → 放量突破MA20
   → 记录突破价、MA20、止损点

2️⃣ 追踪 (次日-第5日)
   → 观察后续4-5日走势
   → 等待买入信号（路径A或B）

3️⃣ 买入 (当天或回踩)
   → 放量涨 → 立即买1/3
   → 缩量企稳 → 立即买1/3
   → 回踩企稳 → 立即买1/3

4️⃣ 加仓 (3-5天后)
   → 站稳MA20 → 加仓1/3
   → 创新高 → 加仓1/3

5️⃣ 止损
   → 3日内跌回MA20下 → 止损全部
   → 或设定下方止损点
```

## ⚠️ 风险提示

1. **虚假突破** — 不是所有突破都能成功，需要追踪确认
2. **系统性风险** — 单个股票突破不代表大盘安全，注意指数
3. **成交量衰竭** — 放量突破后需要成交量维持，否则容易失败
4. **快速获利了结** — 强势股可能快速拉升后回踩，需要耐心
5. **资金管理** — 严格遵守买入1/3、加仓1/3、止损的纪律

## 📞 常见问题

**Q: 为什么有的突破信号失败了？**
A: 突破只是信号，不是100%成功。系统设计就是通过追踪来确认真假，只有追踪后确认有买入信号才真正操作。

**Q: 放量突破后立即下跌怎么办？**
A: 这是"路径B"——缩量回踩。正常的。等待企稳信号（十字星/小阳线+站上MA20）后再介入。

**Q: 资金流验证失败怎么办？**
A: 说明主力资金未跟进，风险较高，可以谨慎参与或观察更多确认。

**Q: 可以改成 MA5 或 MA30 吗？**
A: 可以。修改 `TRACK_DAYS = 5` 为任意周期，但 MA20 是最常用的中期均线，建议保持。

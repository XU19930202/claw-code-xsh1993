# 🤖 飞书 Webhook 配置指南

## 📋 快速步骤（3分钟）

### 第一步：创建飞书应用
1. 打开飞书开放平台：https://open.feishu.cn
2. 左侧菜单 → **应用开发** → **创建应用**
3. 选择 **自建应用** → 输入应用名称（如"周期股监控"）
4. 点击创建

### 第二步：配置机器人权限
1. 在应用详情页 → **权限管理**
2. 搜索并添加以下权限：
   ```
   - im:message:send_as_bot (发送消息权限)
   - chat:read (读取群信息)
   ```
3. 点击保存

### 第三步：获取 Webhook 地址
1. 应用详情页 → **事件订阅** 左侧菜单 → **机器人**
2. 或者在 **开发配置** 中找到 **Webhook 地址**
3. 复制完整的 Webhook URL

### 第四步：配置到脚本
将 Webhook 地址填入：

```python
FEISHU_WEBHOOK = "https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxxxxxxx"
```

## 🔧 高级配置选项

### 选项1：只在有信号时才推送
编辑脚本中的推送条件：

```python
# 仅当信号等级 >= 1 时推送
if signal_level >= 1:
    send_to_feishu(message)
```

### 选项2：推送到特定群组
1. 在飞书中创建一个群组（如"投资监控"）
2. 将机器人加入该群组
3. 系统会自动推送到该群组

### 选项3：自定义推送时间
编辑自动化任务的时间表：

```bash
# 现在的配置：每天下午 5:30
30 17 * * 1-5

# 改为每天下午 6:00 推送：
0 18 * * 1-5

# 改为工作日 9:30 开盘和下午 3:00 推送：
30 9 * * 1-5
0 15 * * 1-5
```

## 🐛 常见问题排查

### 问题1：收不到消息
- ✓ 检查 Webhook 地址是否复制正确
- ✓ 检查机器人是否已加入群组
- ✓ 检查飞书应用权限是否完整

### 问题2：消息格式乱码
- ✓ 确保脚本编码是 UTF-8
- ✓ 检查飞书是否支持该字符集

### 问题3：自动化任务不推送
- ✓ 检查自动化任务是否为 ACTIVE 状态
- ✓ 查看 WorkBuddy 自动化日志
